# Registration Process

## Overview

Registration is handled by Laravel Fortify, with the frontend built in Vue 3 via Inertia.js. The process runs through five stages: rendering the form, submitting it, validating the input, creating the user, and redirecting after success.

---

## Step 1 — Displaying the Registration Form

When a guest visits `/register`, Fortify's `RegisteredUserController@create` is called. Your `FortifyServiceProvider` has told Fortify to respond with an Inertia render:

```php
Fortify::registerView(fn () => Inertia::render('Auth/Register'));
```

Inertia returns the Vue component at `resources/js/Pages/Auth/Register.vue` as a full page response.

---

## Step 2 — The Frontend Form (`Register.vue`)

The Vue component uses Inertia's `useForm` helper to manage four fields:

```js
const form = useForm({
    name: '',
    email: '',
    password: '',
    password_confirmation: '',
})
```

When the user submits the form, Inertia sends an XHR `POST` to the `register` route. On completion (success or failure), the password fields are cleared:

```js
const submit = () => {
    form.post(route('register'), {
        onFinish: () => form.reset('password', 'password_confirmation'),
    })
}
```

Inertia handles CSRF automatically and keeps the page reactive — no full page reload occurs.

---

## Step 3 — Fortify Receives the POST

Fortify's `RegisteredUserController@store` receives the request and delegates to the action class bound in your `FortifyServiceProvider`:

```php
Fortify::createUsersUsing(CreateNewUser::class);
```

---

## Step 4 — Validation and User Creation (`CreateNewUser.php`)

`App\Actions\Fortify\CreateNewUser::create()` runs a validator against the submitted input:

| Field | Rules |
|-------|-------|
| `name` | required, string, max 255 |
| `email` | required, valid email format, max 255, unique in `users` table |
| `password` | required, meets `Password::default()` complexity rules, confirmed (must match `password_confirmation`) |

If validation fails, Inertia returns the errors to the form and each field's `InputError` component displays the relevant message — the user stays on the registration page.

If validation passes, the user record is created:

```php
return User::create([
    'name'     => $input['name'],
    'email'    => $input['email'],
    'password' => Hash::make($input['password']),
]);
```

The password is hashed with `Hash::make()` (bcrypt by default) and never stored in plain text. Emails are lowercased before saving (`'lowercase_usernames' => true` in `config/fortify.php`).

---

## Step 5 — Post-Registration

After the user is created, Fortify:

1. **Fires the `Registered` event** — this triggers Laravel's built-in listener that sends the email verification notification (because `Features::emailVerification()` is enabled).
2. **Logs the user in automatically** — no separate login step is required.
3. **Redirects to `/dashboard`** — the home path configured in `config/fortify.php`:

```php
'home' => '/dashboard',
```

Because email verification is enabled, routes protected by the `verified` middleware will redirect the user to `/email/verify` until they confirm their address.

---

## Summary Flow

```
GET /register
  → FortifyServiceProvider tells Fortify to render Auth/Register via Inertia
  → Vue form displayed to user

User fills in name, email, password, password_confirmation and submits

POST /register
  → Fortify's RegisteredUserController receives the request
  → Calls CreateNewUser::create()
    → Validates input (fails → errors returned to form)
    → Creates User with hashed password
  → Fires Registered event (sends verification email)
  → Logs user in
  → Redirects to /dashboard
```
